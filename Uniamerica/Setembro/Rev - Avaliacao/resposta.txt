04 - Os métodos são definidos diretamente dentro do corpo da classe sem usar a palavra-chave "function". O protótipo de classe evite que cada instancia cire uma copia dos metodos, economizando memoria. Assim todas as intancias compartilham as mesmas funcoes do protótipo.

05 - "justify-content" {

    flex-start (padrão) → itens encostam no início do eixo (à esquerda em row).

    flex-end → itens encostam no final do eixo (à direita em row).

    center → itens agrupados no centro do eixo.

    space-between → primeiro item no início, último no fim, e o espaço restante distribuído igualmente entre os itens.

    space-around → espaço igual ao redor de cada item (ou seja, o espaço nas bordas externas é metade do espaço entre os itens).

    space-evenly → todo o espaço é distribuído de forma igual, inclusive nas bordas externas.
}

10 - O "flex-direction" define qual vai ser o eixo principal do container flexível {
    row -> Eixo principal: horizontal (da esquerda para a direita).
    row-reverse -> Também horizontal, mas os itens aparecem da direita para a esquerda.
    column -> Eixo principal: vertical (de cima para baixo).
    column-reverse -> Vertical, mas invertido: os itens aparecem de baixo para cima.
}

14 - O comando git add . deve ser usado depois que você terminou uma tarefa ou um conjunto de alterações lógicas e antes de você usar git commit para registrar essas alterações no histórico do projeto. Ele é o passo crucial para "selecionar e empacotar" o que será salvo. {
    git add .
    git commit -m "descricao"
}

15 - Quando a funcionalidade está pronta e testada na sua máquina, você executa git push para enviar todo esse trabalho para o reposit…main do projeto no GitHub. {
    push: O comando para enviar.

    origin: O nome padrão para o repositório remoto principal (geralmente o seu repositório no GitHub).

    main: O nome do "ramo" (branch) local que você quer enviar para o ramo de mesmo nome no servidor remoto.
}
